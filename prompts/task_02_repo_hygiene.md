# Task 02 — Repository Hygiene

Create initial repository hygiene files for a Unity take-home project.

## Goal

Prepare the repository before Unity implementation starts.

## Required files

- `README.md` skeleton
- `CREDITS.md` skeleton
- `.gitignore` for Unity
- `docs/folder-conventions.md`
- `docs/first-commit-checklist.md`

## Constraints

- Do not add gameplay features.
- Do not create Unity scripts yet.
- Do not import assets yet.
- Keep the docs aligned with `scope.md`.
- Keep the project focused on a 25-hour IdleOn-inspired vertical slice.

## Expected result

The repository should be ready for Unity project creation, asset compliance tracking, clean commits, and Codex/Cursor-assisted implementation.

## Acceptance test

- README explains project purpose, scope, reviewer path, architecture plan, asset rules, and submission checklist.
- CREDITS includes approved-source asset tracking templates.
- .gitignore excludes Unity-generated folders and local scratch files.
- Folder conventions define the intended Unity structure.
- First commit checklist defines pre-implementation readiness.
